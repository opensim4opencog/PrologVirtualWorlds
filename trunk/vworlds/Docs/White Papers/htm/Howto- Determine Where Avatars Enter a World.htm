How To: Determine Where Avatars Enter a World

by Reena Kawal, Microsoft Research

version 01.00.00.0001, 09/24/99

 

This document addresses the proper use of object properties that determine where avatars enter a world or room on first login and when they move through portals.

 

First, you must specify either a default room or a default portal in the world.  Otherwise, there won't be a place for an avatar to enter.  Specifiying a default room is simplest, but specifying a default portal allows you to define not only an initial position, but an initial orientation and a radius in which avatars will be scattered (so they don't appear to come in on top of each other.)

(1)

          World.DefaultRoom = EntryRoom

or

(2)

          World.DefaultPortal = EntryPortal

 

To place an avatar entering the DefaultRoom as in (1):

Set the room's DefaultAvatarPosition property.  This property is a vector and is set to (0, 0, 0) by default.

 

          EntryRoom.DefaultAvatarPosition = World.CreateVector(-2.1, 0.0, 3.5)

 

To place an avatar entering through a DefaultPortal as in (2), you set the entry point, or position, the entry orientation (the direction the avatar will be facing), and the entry radius:

 

          EntryPortal.EntryPoint = World.CreateVector(-2.1, 0.0, 3.5)

          EntryPortal.EntryOrientation = World.CreateVector(0.0, 0.0, 1.0)

          EntryPortal.EntryRadius = 1

 

Note: you can also use the vector "Set" method to set these properties:

          EntryPortal.EntryPoint.Set -2.1, 0.0, 3.5
