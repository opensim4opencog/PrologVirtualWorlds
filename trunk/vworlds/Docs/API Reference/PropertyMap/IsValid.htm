IsValid

 Method of PropertyMap

Returns True if the string specified by the  bstrName parameter exists as a property in a PropertyMap collection. IsValid is a member of the PropertyMap collection.

Syntax

object.IsValid(bstrName)

The IsValid method syntax has these parts:
Parameters Type Use Description
object PropertyMap Required An object reference to a PropertyMap collection.
bstrName String  Required The name of the property in the PropertyMap collection.

Return Value

This method returns True if the specified property exists in the property map, otherwise, it returns False . 

Remarks

When you use the IsValid method, it searches the specified property map. A PropertyMap object contains an unordered list of paired keys and values, known as key/value pairs. Once you have inserted a key/value pair (element) into a map, you can efficiently retrieve or delete the pair using the key to access it. You can also iterate over all the elements in the map.

A key/value pair maps unique keys to values. The key is a string that represents the name of the property in the property map. The value is the contents of the property. A property map uses the string that makes up the key instead of an index number to access the value of each property.

Example

The first example checks whether a property exists in a property map. 

Sub CheckProperty()
Dim myCDs

Set myCDs = CreatePropertyMap

myCDs.Property("Vocal") = "Ella Fitzgerald"
myCDs.Property("Saxaphone") = "Louie Armstrong"

'The next statement returns True. 

Trace myCDs.IsValid("Vocal")

End Sub
