LightEnabled

 Property of the Light Exemplar


A flag that indicates whether the effects of a Light object are visible. LightEnabled is a read/write Boolean.

Syntax

object.LightEnabled = BooleanValue

The LightEnabled property syntax has these parts:
Parts Type Use Description
object Light Required An instance of the Light Exemplar.
BooleanValue Boolean Required

True: The lighting effects of the light object are visible in the 3-D view. The default setting is True.

False: The lighting effects are not visible.

Remarks

The LightEnabled property is particularly useful when trying to isolate specific lighting effects on surfaces that are illuminated by multiple lights. The cumulative characteristics of light are reflected by the behavior of Light objects in Microsoft Virtual Worlds. If an object shares multiple light sources, the effects of each light are mathematically combined with other lights that share the same space. This combination of light can create unexpected colors and peculiar lighting effects.

To narrow down which light is causing an unexpected effect, the LightEnabled property of each Light object provides a way to systematically shut down lights until the source of the effect is identified.

Example

This example creates a Light object, changes the LightEnabled property to False, and then verifies that the effect is disabled by displaying the status in the Command Window.

'Create a Light object. 

Dim myLight
Set myLight = World.CreateInstance("Light1", "Light", User.Container)

'Disable the light effect using the LightEnabled property. 

myLight.LightEnabled = False

'Trace the current status of the light to the Command Window. 

Trace Light1.LightEnabled
