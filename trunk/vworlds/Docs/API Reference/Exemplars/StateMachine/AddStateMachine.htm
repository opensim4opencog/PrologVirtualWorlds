AddStateMachine

 Method of the StateMachine Exemplar

Adds an additional internal state machine to an existing StateMachine object. AddStateMachine is a member of the StateMachine exemplar.  

Syntax

object.AddStateMachine  bstrStateMachineName, bstrInitialValue

The AddStateMachine method syntax has these parts:
Part Type Use Description
object AddStateMachine Required An object reference that accesses a StateMachine object.
bstrStateMachineName String Required The name of the new internal state machine.
bstrInitialValue String Required The initial state of the added state machine.

Return Value

This method does not return a value.

Remarks

Usually, StateMachine instances are created with a single internal state machine (named Default). For certain applications, however, it may be desirable to have multiple internal state machines, where each internal state machine represents a different attribute of the object. In such cases, you must use this method after a StateMachine object has been created, to add it to another StateMachine.

AddStateMachine automatically adds the newly created internal state machine name to the StateMachines property list.

This method runs on either the client or the server.

To find a state machine's current state, use the Trace method of World, and pass it the name of the state machine object and the internal state machine for which you want to find the state.  

Example

MyStateMachine.AddStateMachine "Flyer", "Waiting"
