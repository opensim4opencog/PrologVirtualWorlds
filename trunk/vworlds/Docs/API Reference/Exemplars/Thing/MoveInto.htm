MoveInto

 Method of the Thing Exemplar

Moves the calling object into a new container. MoveInto is a member of the Thing exemplar.  

Syntax

object.MoveInto pNewContainer

The MoveInto method syntax has these parts:
Part Type Use Description
object Thing Required An object reference that accesses a Thing object.
pNewContainer Thing Required The container that the object is to be moved into.

Return Value

This method does not return a value.

Remarks

MoveInto attempts to move the object from its current container into NewContainer. For this to happen successfully, AllowLeave on the object's old container must return True (the default), and AllowEnter on the object's new container must also return True (the default).

If the CanChangeContainer property of the object to be moved is False, the object can only be moved by the object's Owner (the Avatar who created the object), a Wizard, or a Portal. When the object's IsTakeable property is True, the object can be moved by anyone if there are no other restrictions.

Avatar objects cannot be moved into objects that are not rooms, or rooms that are Null.  

Example

To determine the owner of an object named Art1, execute the following VBScript code:

Trace User.Container.FindWithin("Art1").Owner.Name

The following code will create two artifacts - named Inner and Box (InnerObj and BoxObj are references to these objects). Setting CanChangeContainer on an object determines whether or not that object can be moved to another container.

Dim InnerObj, BoxObj
Set InnerObj = CreateInstance("Inner","Artifact",User.Container)
Set BoxObj = CreateInstance("Box","Artifact",User.Container)

'Move can occur. 

InnerObj.CanChangeContainer = True
InnerObj.MoveInto OuterObj
