SetNamedMeshColor

 Method of the Thing Exemplar

Sets a given mesh to a given color. SetNamedMeshColor is a member of the Thing exemplar.  

Syntax

object.SetNamedMeshColor bstrMeshName, newColor

The SetNamedMeshColor method syntax has these parts:
Part Type Use Description
object Thing Required An object reference that accesses a Thing object.
bstrMeshName String Required The name of the mesh.
newColor Long Required The new color for the mesh.

Return Value

This method does not return a value.

Remarks

When you use this method to set the color of a named mesh, the change applies to every mesh with that name on the object.

The newColor parameter must be set to a Long value in decimal form. To understand the newColor parameter used by SetNamedMeshColor, it is useful to know the internal hexadecimal (base 16) format this method uses. In this format, a color is represented by three pairs of hexadecimal digits in the form &HBBGGRR. Each pair of digits can hold any of 256 values, from &H0 to &HFF (0 to 255). The left-most pair of digits represents the amount of blue in the color, the middle pair represents the amount of green, and the right-most pair represents the amount of red. The entire number can represent any of 16,777,215 distinct colors.

The following table lists the internal hexadecimal color representation for three colors, and the value that should be used for the newColor parameter. By adjusting each color level, you can get the exact color you want.

Color Hexadecimal Value newColor value
Pure Red &H0000FF 255
Pure Green &H00FF00 65280
Pure Blue &HFF0000 16711680
 

Example

MyThing.SetNamedMeshColor "Leg1", 65280
