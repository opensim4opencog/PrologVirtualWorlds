CreateExemplar

 Method of World

Creates a new exemplar that inherits from an existing exemplar. CreateExemplar is a member of the World object.  

Syntax

object.CreateExemplar(bstrType, varExemplar)

The CreateExemplar method syntax has these parts:
Part Type Use Description
object World Required An object reference that accesses a World object.
bstrType String Required The name for the new exemplar type.
varExemplar Variant Required The exemplar to be used as the parent for the new exemplar.

Return Value

This method returns a reference to an instance of the Thing exemplar, unless the Thing core object is specified by the varExemplar parameter (see Remarks).

Remarks

The new exemplar that is created inherits all the properties and methods of its parent exemplar. After the new exemplar is created, additional properties and methods can be added by using the AddMethod and AddProperty methods of the Thing object.

The varExemplar parameter can be either a reference to an exemplar (as may be gotten by the Exemplar property of World), or a String containing the name of an exemplar. The string you use for the varExemplar parameter must be distinct from all other exemplar names. No two exemplars can have the same name.

You can create a root exemplar (one with no parent exemplar) by setting the varExemplar parameter of CreateExemplar to Nothing. In this circumstance, the exemplar that is created inherits from the Thing core object, rather than the Thing exemplar, the usual case. (The Thing exemplar inherits from the Thing core object, but the Thing exemplar has been extended to support additional multimedia features.) Because a root exemplar does not inherit from the Thing exemplar, it has no multimedia properties (such as GeometryName and Sound). The most important reason for creating a root exemplar is to implement an inheritance tree of exemplars, where none of the exemplars needs the multimedia support that Thing normally provides. For example, you can create a 2-dimensional world that does not exhibit multimedia behavior, but behaves more like a dynamic World Wide Web page. In this case, the existing Thing exemplar includes many features that are not necessary to a basic 2-D client interface.

Note: Once you have created an exemplar, there is no way to remove it from the world.  

Example

The two examples that follow show how the second parameter can be either an object reference or a String. Both examples do the same thing.

Set NewExemplar = World.CreateExemplar("SubArtifact1", World.Exemplar("Artifact"))
Set NewExemplar = World.CreateExemplar("SubArtifact2", "Artifact")
