Caller

 Property of World

The Thing that has invoked the current method. Caller is a read-only Thing.  

Syntax

object.Caller

The Caller property syntax has these parts:
Part Type Use Description
object World Required An object reference that accesses a World object.

Remarks

The Caller reference is used to determine, within the body of a method, which object invoked that method. This can be useful if a particular method requires access to the properties (or other methods) of the object that called it.

If a method is invoked directly from the Command Window, or by a user interface action (such as being invoked from a menu item on an object), or by a system message between the server and a client, Caller contains the same value as User. If, however, a method is invoked by another method on a different object, Caller contains a reference to that other object.

To determine the object on which the method is executing (as opposed to the object that is calling the method), see the This property of World.  

Example

The Caller reference will have a meaningful value only if it appears within the body of a subroutine. 

Sub MyRoutine
  Dim Who
  Set Who = World.Caller 
  ' ... Additional code
End Sub
