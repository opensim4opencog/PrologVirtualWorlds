SaveDatabase

 Method of World

Saves checkpoint files. SaveDatabase is a member of the World object.  

Syntax

object.SaveDatabase 

The SaveDatabase method syntax has these parts:
Part Type Use Description
object World Required An object reference that accesses a World object.

Return Value

This method does not return a value.

Remarks

To invoke this method you must be the owner of the world, or a wizard. The SaveDatabase method can only be run on the server.

Checkpoint files are Microsoft Virtual Worlds internal files that have the .vwc extension. Virtual Worlds uses a series of log files to monitor changes to a world. The SaveDatabase method is periodically invoked internally to combine these changes into a single checkpoint file. This "compression" of log files can improve performance factors such as world loading speed.  

Example

World.SaveDatabase
