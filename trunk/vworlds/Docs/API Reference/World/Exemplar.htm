Exemplar

 Property of World

Returns a reference to the exemplar specified in the parameter. Exemplar is a read-only String.  

Syntax

object.Exemplar(stringExemplar)

The Exemplar property syntax has these parts:
Part Type Use Description
object World Required An object reference that accesses a World object.
stringExemplar String Required The type of exemplar, expressed as a String.

Remarks

The Exemplar property of the World object returns an object reference to the exemplar specified in the stringExemplar parameter. This is particularly useful when you want to add or remove a property to or from an exemplar from which all instances of that exemplar inherit from. For example, if you create a custom exemplar named "MyArtifactExemplar", and you want all subsequent instances of that exemplar to have a "middleName" property, you can acquire a reference to that exemplar using the Exemplar property, and then add the "middleName" property to the exemplar itself. The "middleName" property is then inherited by all instances of that exemplar.

If an existing instance of an exemplar has not changed the default value of a property, changing the property at the exemplar level will change the property on the existing instance as well. However, if the property value on the instance has been changed from the default value, Microsoft Virtual Worlds defaults to the changed value on the instance. In this case, changes to the exemplar have no affect on the customized instance. Likewise, if a property is removed from an exemplar, the property is removed only from those instances that have not changed the default value.

Note:   When changes are made to an exemplar object, these changes should be done in a module (as opposed to in the Command Window). Storing your changes in a module assures that the changes are kept when a world file is saved using the SaveToScript method.

Example

This example creates a custom exemplar named "myArtifactEx", obtains a reference to that exemplar, and adds a property to that exemplar. The example also creates an instance of that custom exemplar and verifies that the new property has been added.

'Create a custom exemplar. 

Sub CreateMyExemplar()
Dim myEx

Set myEx = World.CreateExemplar("myArtifactEx", "Artifact")

End Sub


Sub ChangeMyExemplar()

Dim myArtEx
Dim myInstance

'Obtain a reference to the exemplar using
the Exemplar property. 

Set myArtEx = World.Exemplar("myArtifactEx")

'Add a new property. 

myArtEx.AddProperty "Age", 28, PSBIT_FRIENDLY

'Create an instance of the custom exemplar. 

Set myInstance = World.CreateInstance("myArt1", "myArtifactEx", User.Container)

'Verify that the property was inherited. 

Trace myInstance.Age

End Sub
