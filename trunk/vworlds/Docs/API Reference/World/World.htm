World

Object

The World object is a reference to the current world of the user in Microsoft Virtual Worlds. It is a container for all of the other objects in the world.

Scripting in a Virtual Worlds Environment

Normally, any Virtual Worlds method or property you use must be prefaced by the object to which it belongs. For example, a call to a method must take the form of object.MethodName. The exception to this rule is the World object. Any method or property that is not prefaced by an object is assumed to belong to the World object. Therefore, you do not need to preface methods or properties with the World object when scripting from within a Virtual Worlds environment. The following statements have the same result.

mySound = World.CreateSound("Bird.wav", True, True)
mySound = CreateSound("Bird.wav", True, True)

Scripting for Virtual Worlds Using a Browser

If you are scripting with Microsoft Internet Explorer or another browser, you must preface the methods or properties for a Virtual Worlds environment with the World object. To access the World object from a browser, you must first reference the VWClient object. The following code statements create references to a client, the world running on the client, and the user.

Set myClient = Top.myClientObject.VWClient
Set myWorld = myClient.World 
Set myUser = myWorld.User

Creating Scripts With the World Object

The World object is a top level object in the hierarchy of Virtual Worlds. This means that many of the basic methods and properties reside here, such as CreateExemplar, CreateFrame, or SaveDatabase. The default templates such as DefaultAvatarExemplar or DefaultSpriteFile also reside here. The following statement creates a new exemplar.

Set myNewExemplar = World.CreateExemplar("ThunderCloud", _
  "Artifact")

You can also use the MakeHost and UnMakeHost methods to add host capabilities to the specified avatar as shown in the following statement.

World.MakeHost myAvatar
