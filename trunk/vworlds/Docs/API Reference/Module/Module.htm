Module

Object

The Module object exists to provide Microsoft Virtual Worlds scripters with a single, convenient receptacle for one or more script subroutines or functions.  

Using the Module Object

The Module object encapsulates two pieces of information:

  • A flag that indicates whether the module contains methods that will run on the client or on the server.
  • The name of the module.

To create a module, use the CreateScriptModule method.

Set myScript = World.CreateScriptModule("myScript", _
  "myScript.vbs", MODULE_CLIENT + MODULE_SERVER)

The Flags method returns the flags that were used when the module was created.

myModuleFlags = World.Module("myScript").Flags

You can use the ModuleName method to return the name of a module.

myModule = World.Module("myScript").ModuleName

The Install method

If the code that the module refers to contains a subroutine called Install, that subroutine is automatically run on the server when the module is loaded.

The syntax for the Install method is:

    Sub Install(module)
    'Insert your code here
    End Sub

The required module parameter is a reference to the recently created or reloaded module.

One common use of the Install method is to create a custom exemplar from a script module. Within the Install subroutine, a custom exemplar can be created, methods and properties added, and existing properties customized to apply to subsequent instances of that exemplar. For more information on creating a custom exemplar, see the Creating Custom Exemplars topic in the Programmer's Guide.

Note:   Creating instances of a custom exemplar within the Install subroutine can produce unexpected results.
