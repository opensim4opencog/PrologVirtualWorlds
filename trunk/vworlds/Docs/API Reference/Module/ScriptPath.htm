ScriptPath

 Property of the Module Object

Returns the location of a Module that has been added to a world. ScriptPath is a read-only String.  

Syntax

object.ScriptPath

The ScriptPath property syntax has these parts:
Part Type Use Description
object ScriptModule Required An expression that evaluates to a Module object.

Remarks

The ScriptPath property returns the file location of a module that has been added to a world. The value of the ScriptPath property is set when the module is added to the world, so if the file location changes, the ScriptPath property is not updated.

To obtain a reference to a module that has been added to a world, use the Module property of the Method object.

Tip:   The recommended way to move a module that has been added to a world is to delete the module from the world, and then add it back using the new path. This method ensures that the ScriptPath property is updated, and the references to members of the module are pointed at the correct file location.

Example

This example displays the file location of a module that has been added to a world. The parameter "myStrModuleName" is the String name of a valid module. 

Sub myTraceModulePath(myStrModuleName)
Dim myFlag
Dim myModuleObject

'Verify that the string passed is a valid module name. 

myFlag = World.IsValidModule(myStrModuleName)

'If the module is valid, acquire a reference to it and
'display the ScriptPath in the Command Window. 

If myFlag = True Then
    Set myModuleObject = World.Module(myStrModuleName)
    Trace myModuleObject.ScriptPath
Else
    Trace myStrModuleName & " is not a valid module."
End If

End Sub
