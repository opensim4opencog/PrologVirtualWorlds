Flags

 Property of the Module Object

A flag that indicates whether the code contained in the script module can be run on a client or on the server.  Flags is a read-write Long.  

Syntax

object.Flags = flags

The Flags property syntax has these parts:
Part Type Use Description
object Module Required An reference to a Module object.
longValue Long Required A flag value that determines if the code referred to by the Module can be run on the client, on the server, or on both.

Remarks

When you use the CreateScriptModule method to create a script module, you specify whether the module will run on a client, a server, or both by specifying the value for the lFlags parameter.

The flags used for a Module object are made up of combinations of constants with Long values as illustrated in the table below. You can combine these flags by adding them together when a module contains code that should run on both the client and the server.  For convenience, these constants are available as read-only properties of the World object.  

Flag Value Description
MODULE_CLIENT 1 Used for modules that contain methods designed to run on a client.
MODULE_SERVER 2 Used for modules containing methods that are designed to run on the server.
 

Example

To get the Flags property and check for a specific value:

Result = MyModule.Flags
If (Result AND World.MODULE_SERVER) = World.MODULE_SERVER Then
   World.Trace "Script from this module can be run on the server!"
End If

To set the Flags property of a module:

MyModule.Flags = World.MODULE_CLIENT +  World.MODULE_SERVER
