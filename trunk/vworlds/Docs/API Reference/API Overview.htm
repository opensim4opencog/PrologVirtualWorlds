Virtual Worlds API Overview

Understanding the Documentation

The Microsoft Virtual Worlds API Documentation is split into three logical sections: Exemplars, Objects, and Event Handlers. Under each major heading, there are several topics that explain how to access corresponding API features from a Microsoft ActiveScripting Language. Most of the syntax and code examples given are for VBScript, but in most cases, JScript can also be used.

For each object and exemplar, the documentation includes an overview topic as well as topics for each of the properties and methods of the object.

API Syntax

Properties

For both read/write and read-only properties, the object statement is the object that the property is associated with. The type of this object is described in the syntax description table that follows each syntax statement. Properties that are read/write include an assignment operator (=), and the required type described by the typeValue element. Properties that are read-only do not include the assignment.

Read/Write property:

object.Property = typeValue

Read-Only property:

object.Property

Methods

For methods, the object statement is the object that the method is associated with. The type of this object is described in the syntax description table that follows each syntax statement. Methods that have a return value include parentheses around the parameters. If a method has a return value but no parameters, an empty set of parentheses are included. If a method does not have a return value, no parentheses are included.

The arg1 and arg2 statements represent required parameters. brackets surrounding a parameter (the optionalArg1 and optionalArg2 elements) indicate the parameter is optional.

Method with a return value:

object.Method(arg1, arg2 [,optionalArg1] [,optionalArg2], ...)

Method with no return value:

object.Method arg1, arg2 [,optionalArg1] [,optionalArg2], ...

Event Handlers

Event handlers use the VBScript Function tag, which implies a return value. The OnEventHandler statement represents the name of the event handler. The arguments follow the conventions of method arguments, where an element with no brackets is required, whereas an element surrounded by brackets is optional. The statements element is the optional code that may be included in the Function. The return value of the Function is described in the statements section of the syntax description table.

Function OnEventHandler(arg1, arg2 [,optionalArg1] [,optionalArg2], ...)
statements
End Function


Exemplars

Exemplars are template objects used to create instances, or copies, of that object. Each instance of an exemplar inherits all of the properties and methods of the exemplar, but can then be further customized by changing property values or adding or removing properties or methods.

Exemplars can be created from instances of other exemplars. This provides for multiple levels of inheritance in some cases.

Microsoft Virtual Worlds provides a number of preconfigured exemplars to help represent objects you might want in a virtual world such as artifacts, rooms, avatars, etc. All of these provided exemplars inherit from the Thing exemplar, so instances created from them will inherit properties and methods from both the specific exemplar and the Thing exemplar.

For more information on Exemplars, see the Programmer's Guide.

Objects

The objects in Virtual Worlds provide access to a core set of useful system features. Some of them, such as Thing and World, provide much of the basic functionality of the Virtual Worlds platform while others, such as PropertyList, PropertyMap, and Vector, simply provide complex data types that are used throughout the API. Every Virtual Worlds object supports two methods (IsOfType and ToASCII) and two properties (Type and World) that are inherited from a common base class. Also, the Thing exemplar (and therefore all of the provided exemplars) inherits properties and methods from the Thing object.

Event Handlers

In addition to the properties and methods of exemplars and objects in Virtual Worlds, there is a complex event system. Events fire in the world in response to a number of phenomena including data changes, avatar and object movements, avatars speaking, etc. Every time an event fires, the system calls a corresponding event handler or set of event handlers. These handlers almost always exist as specifically-named methods on objects and usually take a set of parameters appropriate for the event being handled. This section of the documentation discusses the purpose and syntactic information for each event handler called by the Virtual Worlds system.

For more information on events, see the Programmer's Guide.
