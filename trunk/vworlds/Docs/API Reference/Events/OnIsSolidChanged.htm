OnIsSolidChanged

 Event Handler

An event-handling method that creates or destroys a boundary. It is called in response to the IsSolidChanged event, which occurs whenever the IsSolid property for an object is changed. Whether a boundary is created or destroyed depends on how the IsSolid property is changed.

Syntax

Sub OnIsSolidChanged(Hint, HintData)
    statements
End Sub

The OnIsSolidChanged method syntax has these parts:
Part Type Use Description
Hint Long Required The type of change that occurred.
HintData Variant Required The new value of the IsSolid property.

Return Value

This method does not return a value.

Remarks

This method is not normally called by the user; it responds to an internal event that occurs in Virtual Worlds when the IsSolid property of an object has changed.

Note:   To ensure that the internal procedures that are tied to each event handler are invoked, you must call InvokeSuperMethod within the overridden subroutine. For more information on using this method, see the InvokeSuperMethod topic.

Example

Sub OnIsSolidChanged (Hint, HintData)
    World.Trace "IsSolid property has changed to: " & CStr(HintData)
    This.InvokeSuperMethod (Array(Hint, HintData))
End Sub
