OnIntersectee

 Event Handler

An event-handling method that is called when another object collides with the calling object's boundary.

Syntax

Sub OnIntersectee(pIntersector, fX, fY, fZ, iEdgeCycleIndex, iEdgeIndex, bTransition)
    statements
End Sub

The OnIntersectee method syntax has these parts:
Part Type Use Description
pIntersector Thing Required The object that has collided with the calling object's boundary.
fX Float Required The x-coordinate of the position of the intersection.
fY Float Required The y-coordinate of the position of the intersection.
fZ Float Required The z-coordinate of the position of the intersection.
iEdgeCycleIndex Integer Required The index of the boundary with the BoundaryList that is intersected.
iEdgeIndex Integer Required The index of the edge within the boundary.
bTransition Boolean Required A flag describing the type of cell intersection.

Return Value

This method does not return a value.

Remarks

The x, y, and z coordinates are relative to the frame containing the cell.

A Transition value of 1 indicates the object attempted to cross a boundary from the outside; a value of 0 indicates that a boundary was exited. See the Boundary core object for more information.

This method is not normally called by the user; it responds to an internal event that occurs in Virtual Worlds on collision by an object with the cell around a Thing. If you find it necessary to override this event-handling method, InvokeSuperMethod should also be called to maintain the existing functionality of the overridden method.

When the Thing that has been intersected is a Room Portal or a Interworld Portal, the portal calls a method that displays a dialog box, taking the URL (Uniform Resource Locator) for the dialog box from the portal's PortalDialogURL property. For this reason, if you plan to override OnIntersectee with your own version, you should include a call to InvokeSuperMethod in your overriding version, to ensure that this dialog box is displayed.

Example

Sub OnIntersectee(Object2, x, y, z, BoundaryIndex, EdgeIndex, Transition)
    World.Trace "The intersecting object was: " & Object2.Name
    This.InvokeSuperMethod(Array(Object2, x, y, z, BoundaryIndex, EdgeIndex, Transition))
End Sub
