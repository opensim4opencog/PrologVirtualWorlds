OnPortalLeave

 Event Handler

An event-handling method that is called in response to the PortalArrive event, which occurs when an object leaves a region through a Portal.

Syntax

Sub OnPortalLeave(ThePortal, LeavingObj, Index)
    statements
End Sub

The OnPortalLeave method syntax has these parts:
Part Type Use Description
ThePortal Portal Required The portal through which the object is leaving.
LeavingObj Thing Required A reference to the object leaving through the portal.
Index Integer Required Whether the object is departing for a primary or secondary destination.

The PortalLeave event supports these additional handlers:
Event Handler Description
OnPeerPortalLeave The handler for other objects in the same Container.
OnContainerPortalLeave The handler for the object's Contents.
OnContentPortalLeave The handler for the Container of the object.

Return Value

This method does not return a value.

Remarks

This method is not normally called by the user; it responds to an internal event that occurs in Virtual Worlds when a file is dropped from the desktop into the world. If you find it necessary to override this event-handling method, InvokeSuperMethod should also be called to maintain the existing functionality of the overridden method.

It is sometimes useful to override this method in order to display a farewell message or trigger other behaviors when a user departs.

The parameter Index indicates whether the destination for which the object is departing is primary or secondary. Primary destinations are indicated with a value of -1. Secondary destinations are indicated with a value greater than -1.

Example

Sub OnPortalLeave(ThePortal, LeavingObj, Index)
    Trace LeavingObj.Name & " is leaving."
    This.InvokeSuperMethod(Array(ThePortal, LeavingObj, Index))

End Sub
