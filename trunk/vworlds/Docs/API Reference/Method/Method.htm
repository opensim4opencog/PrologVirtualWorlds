Method

Object

The Method object is an in-world wrapper for a script subroutine or function that is either an inline module or is contained in a script module.

Using the Method Object

The Method object exists to encapsulate certain pieces of information:

  • Scripting code for an inline method
  • A flag that specifies whether the method is to run on the client, the server, or both, as well as whether the method is to run synchronously or asynchronously
  • The name of the method
  • The module that contains the script subroutine or function associated with non-inline methods

Use the CreateMethod method to create a new method. Unless the method you want to create is an inline method, you must add the method to a script file and add the file as a module as described in the following section Attaching Methods to Objects.

If you want to create more than one procedure or function in one file, you must add those to Microsoft Virtuals Worlds as a module. To create an inline method, use the CreateInlineMethod method.

Attaching Methods to Objects

You can create a method from a script module, and then attach it to objects in Virtual Worlds in order to enable the objects to perform specific actions. After you create a method, you can attach it to an object by using the AddMethod method.
