IsValidMethod

 Method of Thing

Determines whether the specified user-added method exists on either the object or on an exemplar in its inheritance tree. IsValidMethod is a member of the Thing object.  

Syntax

object.IsValidMethod(bstrName)

The IsValidMethod method syntax has these parts:
Part Type Use Description
object Thing Required An object reference that accesses a Thing object.
bstrName String Required The name of the user-added method.

Return Value

This method returns a Boolean.

Remarks

IsValidMethod first looks for the specified method on the calling object. If the method can not be found there, IsValidMethod then looks for it on the object's exemplar. If necessary, IsValidMethod will search for the specified method all the way to the base exemplar for the object. For example, if you use IsValidMethod to determine whether a particular method exists on a StateMachine instance, it may be necessary for IsValidMethod to search the instance, the StateMachine exemplar, the Artifact exemplar, and finally the Thing exemplar.  If the method is found on any intermediate exemplar, IsValidMethod abandons the search and returns True.

To prevent calling a method that does not exist, and the errors that result, first use IsValidMethod to determine that the method exists, and then call it only if it does.

Please note that IsValidMethod will not detect most of the methods built in to the Virtual Worlds system (e.g. checking IsValidMethod("IsValidMethod") will return False).  For this reason, it is most effectively used to check for the existence of methods added by users or world designers.

Example

Result = MyObject.IsValidMethod( "DoSomething" )
If Result = True Then
   MyObject.DoSomething		'Call method only if it exists
Else
   Trace "DoSomething method does not exist."
End If
