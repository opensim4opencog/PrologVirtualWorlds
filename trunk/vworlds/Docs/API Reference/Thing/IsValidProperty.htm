IsValidProperty

 Method of Thing

Determines whether the specified user-added property exists on either the object or on an exemplar in its inheritance tree. IsValidProperty is a member of the Thing object.  

Syntax

object.IsValidProperty(bstrName)

The IsValidProperty method syntax has these parts:
Part Type Use Description
object Thing Required An object reference that accesses a Thing object.
bstrName String Required The name of the user-added property.

Return Value

This method returns a Boolean that indicates whether the specified property exists on the invoking object.

Remarks

IsValidProperty first looks for the specified property on the calling instance of the object. If the property is not found there, IsValidProperty then looks for it on the object's exemplar. If necessary, IsValidProperty will search for the specified property all the way to the base exemplar for the object. For example, if you use IsValidProperty to determine whether a particular property exists on a StateMachine instance, it may be necessary for IsValidProperty to search the instance, the StateMachine exemplar, the Artifact exemplar, and finally the Thing exemplar.  If the property is found on any intermediate exemplar, IsValidProperty abandons the search and returns True.

To prevent accessing a property that does not exist, first use IsValidProperty to determine that the property exists, and then access it only if it does exist.

Please note that IsValidProperty will not detect most of the properties built in to the Virtual Worlds system (e.g. checking IsValidProperty("IsStub") will return False).  For this reason, it is most effectively used to check for the existence of properties added by users or world designers.

Example

Result = AThing.IsValidProperty( "Age" )
If Result = True Then
   Value = AThing.Age		'Access property only if it exists
Else
   Trace "Age property does not exist."
End If
