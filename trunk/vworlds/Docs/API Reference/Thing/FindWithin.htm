FindWithin

 Method of Thing

Searches by name within the invoking object's contents for a specified object. FindWithin is a member of the Thing object.  

Syntax

object.FindWithin(bstrName [, varQuiet])

The FindWithin method syntax has these parts:
Part Type Use Description
object Thing Required An object reference that accesses a Thing object.
bstrName String Required The name of the requested object.
[varQuiet] Variant Optional Equal to 0.

Return Value

This method returns a Thing.

Remarks

The invoking object should be the container that is to be searched to find the requested object.

If FindWithin is successful, it displays OK in the Command Window. If the requested object cannot be found, it displays Failed to find "<ObjectName>" within "<Container>".

When FindWithin is called from the server, it is able to find anything for which it is searching, in the container. When FindWithin is called from a client, however, it may not find every object in the same container, because clients have only a subset of the objects in a world. Specifically, only those objects in the same room as your avatar exist on your client. Such objects as another avatar's contents or the contents of closed containers are not sent to the client, and cannot be found by FindWithin if it is called from the client.

If the object is not being marshalled (for example, if you call this method on the client to find an object that isn't being regularly marshalled to your client), you will get only a momentary, one-time copy of the object. The original object, from which the copy was made, may undergo further changes after you have gotten the copy.  

Example

Set Result = AThing.FindWithin ("SomeObject")
