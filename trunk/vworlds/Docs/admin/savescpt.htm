Exporting a World as a Script File

After you have made changes to your world, you should back up your changes. One way to do this is to export the data to a script file. (The other way is to save the world's data to a checkpoint file). The script file that is produced contains a single Microsoft Visual Basic Script© subroutine named AutoRun. When this script is loaded in the Import Script Wizard, the subroutine re-creates all of the objects in the world exactly as they were at the time the script was exported. Although the script can be edited, it is recommended that do not make changes in this file unless you are sure of their effects. The script file contains Microsoft Visual Basic Script© statements that are sensitive to minor changes, as are statements in any other programming language.

To export a world as a script file:

  1. In the Virtual Worlds Server application, click the title bar of the world you want to export.
  2. Click the File menu, and then click Export.

The Virtual Worlds Server application writes a Microsoft Visual Basic Script© file containing all of the objects of the world and their methods and properties. This file is written to the Worlds directory. The default path to this directory is C:\Program Files\Microsoft Virtual Worlds\Worlds\.

To export a world as a script file, but with a different name:

  1. In the Virtual Worlds Server application, click the title bar of the world you want to export.
  2. Click the File menu, and then click Export As....
  3. When the Export As dialog box appears, type the new filename in the File name input box.
  4. Click Save.

The Virtual Worlds Server application writes out a Microsoft Visual Basic Script© file to the Worlds directory, using the name you entered. This file contains all of the objects and their methods and properties.
