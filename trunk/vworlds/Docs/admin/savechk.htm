Saving a World as a Checkpoint File

After you have made changes to your world, you should save your changes. One way to do this is to save the world as a checkpoint file. (You can also save the world in a script file.) A Virtual Worlds checkpoint file is a computer-readable database file that contains all objects in the world, together with all of their methods and properties. A checkpoint file has a .VWC extension.

After you have saved the world, you can restore the world to its exact state when it was saved. Do this by clicking the Open... menu option on the Virtual Worlds Server application's File menu.

Important! You can Open... a checkpoint file only if it was saved by the same version of the Virtual Worlds Server application. If you plan to upgrade to a new version of Virtual Worlds, you need to Export the world as a script file in the old version. After you have upgraded to the new version of Virtual Worlds, you can then Import the saved script file. For details, see Exporting a World as a Script File.

To save a world as a checkpoint file:

  1. In the Virtual Worlds Server application, click the title bar of the world you want to save.
  2. Click the File menu, and then click Save.

The Virtual Worlds Server application writes a checkpoint file containing all of the objects of the world as well as their methods and properties. This file is written to the Worlds directory. The default path to this directory is C:\Program Files\Microsoft Virtual World\Worlds\.

To save a world as a checkpoint file, but with a different name:

  1. In the Virtual Worlds Server application, click the title bar of the world you wish to save.
  2. From the File menu, click Save As....
  3. When the Save As dialog box appears, type the new filename in the File name input box.
  4. Click Save.

The Virtual Worlds Server application writes a checkpoint file to the Worlds directory, using the name you entered. The default path to this directory is C:\Program Files\Microsoft Virtual World\Worlds\.
