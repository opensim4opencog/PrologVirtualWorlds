Moving a World

A Virtual World consists of a meta-database and graphic, sound, and other content. The database, or checkpoint file, contains information about all the objects in the world. Content may be located on each Virtual Worlds client, or it may be accessed remotely by URL (Uniform Resource Locator) from a shared directory on another computer on an intranet, or from a Web server. The world's database, or checkpoint file, and its content do not need to reside on the same computer.

If necessary, you can move a Virtual World from one server to another. The most common reason to do this is when you create content on one computer (often local) and publish the world and its content to another. This operation requires you to move the world's checkpoint, and any content that may be on that computer. The first part of this topic provides information about what to do when you move a world's data to another server; the second part provides details on how to move (or publish) world content from one computer to a Web server using the Microsoft Web Publishing Wizard.

The most important point to keep in mind when you move a world to a new server is that the directory structure must be left intact. If it is not, clients logging on to the new server are not able to locate the content they need.

To move the world's data to a new server:

When the world's data and its content are located on different computers

  1. Copy the checkpoint file to its new destination.
  2. On the destination computer, click the Virtual Worlds Server application's File menu option, and then click Open... to open the checkpoint file on the new server.

When the world's data and its content are located on the same computer, and both are to be moved

  1. Copy the checkpoint file to the new server.
  2. Click the Virtual Worlds Server application's File menu option, and then click Open... to open the checkpoint file on the new server.
  3. Change the RootURL to include the new content path. Note that to change World properties, you must log on to the world using the same avatar that you used to create the world.

As an alternative to moving the world's checkpoint file, you can export the world as a script file, copy the script file to the new server, and then import the file using the Import Script Wizard in the Virtual Worlds Server application. If you also intend to move the world's content, however, you must include the new path to this content in the RootURL property, as detailed in the previous paragraph.

To publish content to a Web server:

Before you can publish world content to a Web server, you need to contact the server administrator to get an account on it. The server administrator will set up a directory for your use, assign you a user name and password on the server, and set appropriate write permissions for you on that server.

Once you have been given an account on the server, you can then transfer files to the server. Although there are several ways that this can be done, this focus of this topic is on transferring files using FTP (File Transfer Protocol). If you are familiar with FTP, you may use any FTP client and skip the rest of this section.

To start the Web Publishing Wizard, first click the Start menu, and then click Programs, then click Internet Explorer, and then click Web Publishing Wizard. If you do not find the icon for the Web Publishing Wizard, it may not be installed. You can download the Web Publishing Wizard from this site: http://www.microsoft.com/ie/

  1. When the Publish Your Files on the Web Wizard page appears, read the introductory information contained in this page, and then click Next.
  2. On the Select a File or Folder Wizard page, enter the file or folder name of the files you want to publish to the Web server. As an alternative, you can click the Browse Folder... button to locate the directory you want to publish to the server. The Browse Files... button displays the files within that directory. When you have located the directory you want to publish, click Next.
  3. On the Name the Web Server Wizard page, enter a name to describe your Web server. You can type any name you want, but it is helpful to use a name that you can easily remember. After you have entered a name for the Web server, click Advanced.
  4. On the Select Your Service Provider Wizard page, choose FTP from the options available in the select option box, and then click Next.
  5. On the Specify the URL and Directory Wizard page, enter the URL (Uniform Resource Locator) or Internet address of the Web server hosting your web pages. The URL or Internet address may look something like this: http://www.microsoft.com/myname. The other text input box on this page should contain the local directory (on your computer) that you are publishing to the Web server. Click Next.
  6. On the Specify the FTP Server and Subfolder Wizard page, enter the name of your FTP server if it is different from your Web server. If you plan to publish files to a subdirectory in your account area, enter the name of this subdirectory in the Subfolder input box. Click Next.
  7. On the Publish Your Files Wizard page, click Finish. The first time you use the Web Publishing Wizard, you see the Enter Network Password dialog box. Enter the user name given to you by the Web server administrator, and then the password. Note that these can be different from the user name and password you normally use to log on to your network. Click OK.
    The Publishing Files dialog box that displays lists the individual files being uploaded. When all of the files have been uploaded, another dialog box informs you of this. Click OK.
