Importing a World Script File

A world script file consists of a single subroutine containing all the Microsoft Visual Basic Script© statements necessary to re-create a world. The subroutine, which is named AutoRun, is executed when you click Import... on the Virtual Worlds Server application's File menu. Before you can import a world script file, it must first have been exported as a Microsoft Visual Basic Script© file, using either the Export or Export As... menu options on the File menu of the Virtual Worlds Server application.

The most common reason for importing a world script file is to be able to re-create your world after you have upgraded to a newer version of Virtual Worlds. In this case, you would export your world to a script file using the older version of Virtual Worlds, and then import the same script file into the newer version.

To import a world script file:

  1. Click the File menu on the Virtual Worlds Server application, and then click Import....
  2. When the Import VBScript world file window opens, type in the required information:
    • Name of the world
    • Description (optional)
    • Server location - Use localhost for a world hosted on the same computer. Use the network computer name for a world hosted on a different computer
    • Your logon name
    • Password (optional)
    • Confirm password (if you entered a password)
    • Script filename - the Browse button opens a File Explorer to help you find the script.
  3. Click Import.
  4. Click OK.

If your script file loaded successfully, the Script window displays 'Done Creating world "World name" '.
