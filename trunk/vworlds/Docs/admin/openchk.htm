Opening a World Checkpoint File

When you create or save a world, all of the world information is saved in a binary file format called a checkpoint file. You can also save (or export) the world information as a Microsoft Visual Basic Script© file, a text file.

When the Virtual Worlds Server application starts, it automatically opens any checkpoint files contained in the ~/Microsoft Virtual Worlds/Worlds directory. However, checkpoint files added to this directory after the server starts are not automatically detected. Use this menu option if you have added a world checkpoint file after the server starts.

You can also use this menu option to restore a world that has been deleted, providing that the checkpoint backup file still exists. Both a checkpoint file and a backup checkpoint file are created when the world is first created, and each is saved whenever you save your world. For example, if you have created a world named Home, its checkpoint file would be Home.VWC and the checkpoint backup file would be Home.BAK.VWC.

To open a world checkpoint file:

  1. On the Virtual Worlds Server application's File menu, click Open....
  2. When the Open checkpoint file: dialog box appears, select the checkpoint file for the world you wish to open. All of the checkpoint files and backup checkpoint files are shown in the file explorer pane.
  3. Click Open.

Note: Checkpoint files must be placed in the ~/Microsoft Virtual Worlds/Worlds directory. If you attempt to open checkpoint files from any other directory, an error dialog box will be displayed.

To restore a world that has been deleted:

  1. Using Windows Explorer, rename the file Home.BAK.VWC to Home.VWC. (The actual file names will probably be different.)
  2. On the Server Administration tool's File menu, click Open....
  3. When the Open checkpoint file: dialog box appears, select the checkpoint file for the world you wish to open. All of the checkpoint files and backup checkpoint files are shown in the file explorer pane.
  4. Click Open.
