User Authentication

Users connecting to the Virtual Worlds server can be authenticated using the NTLM (Windows NT Lan Manager) authentication scheme. The Virtual Worlds server, when running on Windows NT, can require clients to belong to the same domain if the NTLM Authentication option is turned on (on by default). For most intranet scenarios, users will already be authenticated from their initial log on.

For internet scenarios, or users connecting from outside the domain, they will be presented with the standard Windows NT three-tiered logon dialog box (username, password, domain). Users, not avatars, must be members of the domain the server occupies in order to complete the connection request.

To Toggle User Authentication:

  1. In the Virtual Worlds Server application, click the Tools menu, and then click User Authentication....
  2. Check (or uncheck) the "Require NTLM Authentication for Logins" to turn on (or off) NTLM Authentication.

To Set a Local User Group:

You may also further restrict server access by setting a Local Group within the server machine's domain. If this item is checked, users must belong to that Local Group. On the server machine,

  1. Ensure that you are logged in as an administrator in the local domain
  2. Launch the User Manager from the Administrative Tools Start Menu shortcut (NT4) or Control Panel (Win2K).
  3. Switch to the local domain (to view local user accounts and groups)
  4. If none of the existing local groups are sufficient, create a new local group via the User->Create New Local Group... menu item, say "VWorldsUsers"
  5. Add users from the broader domain to "VWorldsUsers" or create new user accounts on the local domain and add them to the "VWorldsUsers" local group
  6. If you are providing new user accounts, notify those people of their account information
  7. Launch the Virtual Worlds server application, ensure you check the Require Local Group Membership checkbox in the Tools->User Authentication menu item and fill in the text box with "VWorldsUsers"

Note that users logging in will now have to enter the server name as the domain if Local Group Membership is required.
