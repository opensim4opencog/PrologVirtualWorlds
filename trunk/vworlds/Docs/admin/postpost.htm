Adding Custom Content to a Posted World

When you make content changes to a world you have already uploaded, or posted to a Web server, there are things you must do to ensure that clients see those changes. Virtual Worlds relies on finding content in specific locations. This content can be remote content on a Web server or in a shared directory on another computer, or it can be local content on each client. Content on the client computers must be located in the Local Content directory (or in a subdirectory below it) of the Virtual Worlds installation directory. The content author determines where Virtual Worlds finds content by using an URL (Universal Resource Locator) to locate the content or by setting the RootURL property of World.

Posting Custom Avatar Graphics

After you have created a custom avatar that you would like to make available to users in your world, you need to post the avatar graphics to the Web server that is hosting your world's content. The topic Moving a World provides information about moving content from one computer to another.

Making Custom Avatar Graphics Available to Users

Once custom avatar graphics have been posted to the Web server, users must download these files to their computers in order to use of them. This can be done in either of the two ways listed below.

  • Users can select the Auto download new avatar graphics option on the Options Dialog Box.
  • Users can manually download the files from the server to their client computers. They need to be careful to copy these files to the appropriate directory. (The default directory for avatar graphics is C:\Program Files\Microsoft Virtual Worlds\Local Content\Avatar Graphics.)

Posting Other Custom Content

When you create custom content other than avatar graphics, view it locally before you post it to the Web server that hosts your world's content. You can do this by entering the path to the content in the File path text box of the Geometry Property Page (objects with simple graphics) or the Texture Property Page (objects with more complex graphics). When you are satisfied with your work, you can post it to your Web server. Remember that you need to post this file to the same directory on the Web server. Another point to consider is that indirectly referenced content, such as a texture map on a 3D model, also needs to be posted.

If you are creating a lot of content, make one or more new directories for this content. These directories are displayed in the Models selection box in both the Geometry Property Page and the Texture Property Page. To view this content locally, select the appropriate model, and then select the graphics file you want to view from the file list that appears. When you have completed work on this content, post the directory (or directories) to your Web server. As above, use the same directory structure on the Web server as on your local computer.

Content can be stored on any Web server. To associate an object in your world with content on a Web server, enter the URL for the content in the File path input box of either the Geometry Property Page or the Texture Property Page for the object. Here is an example: http://mywebserver/local content/sdk/library/models/animal/ant.x.
