Answers

When I right-click an avatar in the Avatar list or the Room Contents list, the menu I see is different from the menu that appears when I right-click an avatar in the graphical window.

To pop up the same menu in these two lists as in the graphical window, you must click on an avatar with the left mouse button.

When I open the Client Logon page from the Start Menu Shortcut, I get an error dialog box stating that "the object doesn't support this property."
- or -
I get script security warnings when I try to load the logon page or enter a world.

Both of these problems are related to control registration. See the question During installation, I get a message that some components failed to register in the Troubleshooting section of Readme.TXT.

After the spinning cube, I see only avatars and a few other objects, but I don't see any 3D geometry. The whole background is black.

Entering a world and downloading its content are two separate actions. It's likely that, for some reason, the world is looking for its content in the wrong place. It is either looking on your local drive for nonexistent files, or on a web server that is not working. Check with the world administrator/owner/creator.

Some of the graphics have weird colors, but they're not white.

Check your display properties. Try setting the color depth to 16 bit or higher and see if that helps.

When running Virtual Worlds, my graphics window is much smaller than the space allocated for it in the user interface.

You are using too much of your computer's video memory by running it in a high color depth. Try reducing the color depth to a setting below 24 bit color.

I created a large solid object and now I seem to be trapped inside it.

You can either uncheck the object's Is solid property (on the General Property Page for the object), or create another object, move it outside the boundary of the object you are trapped in, and then select Approach the newly created object.

What do I do when I get a message that says my avatar is corrupt?

If an avatar has been corrupted, all you can do is delete it and start over.

I logged on to a Virtual World as "Bob", but when I entered the world, my avatar name was "Bob13".

All avatars in Virtual Worlds must have unique identifying names. If other avatars have already logged on to a world with the name "Bob", Virtual Worlds automatically resolves names of new avatars by adding a number to differentiate them from avatars already in the world.

I can't hear chat from other rooms. What's wrong?

This is normal. A room forms a natural boundary in Virtual Worlds. You will not receive any information about a room unless you are in it, including chat taking place there.

Someone is in the room but I can't hear his/her chat. Why?

Did you add that avatar to your Ignored list? Ignored avatars do not display their text messages in your chat history pane. Try left-clicking one of them in the Avatar list and see if one of the menu items is UnIgnore. If so, click UnIgnore to allow messages from that avatar to reach you.

Someone is bothering me in one of the rooms. What can I do to ignore this person?

In the Avatar list, left-click the avatar who is bothering you, then click Ignore. You will not see any text or gestures from this avatar in the chat history pane. You cannot prevent an avatar from moving toward you in the graphical window.

VBScript error: This page is taking an unusually long time to finish

You may see this error when loading the geometries and properties of certain virtual world environments. Do not click Cancel; if you wait a few more moments, the error message disappears and Virtual World loads.

VBScript Error: Failed to Connect
  • Check to see whether the server application is running. The Virtual Worlds server must always be running in order to connect to and change a Virtual World. If you are running a server locally, check the server's status display to see whether a connection has been made.
  • Can other applications connect to the network, or to the computer running your server?
  • If you are logging on with an existing avatar, are you using the right password?

Other VBScript Errors
  • Check your Internet Explorer safety level. You must set your Internet Explorer safety level to Medium, or several VBScript commands that are required for logging on will not be able to execute.
