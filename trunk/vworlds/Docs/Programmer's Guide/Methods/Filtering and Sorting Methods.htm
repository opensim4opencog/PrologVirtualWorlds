Filtering and Sorting Methods

When you open the Object Explorer, and you select an object in the current room, the Object Explorer displays all methods (and properties) on that object. Some of the displayed methods are inherited from the object's exemplar, while the rest are instance methods that are specific to that object. By filtering the methods, you can specify that only exemplar methods should be displayed, or only instance methods, or both types.

You can also sort methods by name, by the module they belong to, or by the subroutine name within the module.

Method Filtering Options

When you filter methods, you are not deleting any of them; you are telling the Object Explorer not to display the ones that have been filtered out. You can set filtering options separately for properties and methods.

Filtering options persist when you change the selected object and view another object's methods. For example, if you have changed the filtering so that only instance methods are visible, you must change the filtering back in order to see the exemplar methods in any other object in your virtual world. By default, all methods (exemplar and instance) are displayed.

To display only instance methods for an object:

  1. Right-click anywhere in the object's Methods pane.
  2. If a check mark is displayed on Show Exemplar Methods, click this menu item to turn this option off. (Note: if no check mark is displayed, you are already seeing only the instance methods.)

To display only exemplar methods for an object:

  1. Right-click anywhere in the object's Methods pane.
  2. If a check mark is displayed on Show Instance Methods, click this menu item to turn this option off. (Note: if no check mark is displayed, you are already seeing only the exemplar methods.)

To display all methods for an object:

  1. Right-click anywhere in the object's Methods pane.
  2. If either the Show Instance Methods menu item or the Show Exemplar Methods menu item is unchecked, click that menu item to turn that option on. (Note: if neither check mark is displayed, you will need to carry out these steps twice, once for each menu item.)

Method Sorting Options

You can sort an object's methods in any of three ways:

  • by method name
  • by the name of the module that contains it
  • by the name of the subroutine within the module

Each sorting option is done alphabetically. Sorting options persist. When you select another object, its methods will be sorted using the option you selected previously. By default, methods are sorted by name.

To sort methods by name:

  1. Select the object whose methods you want to sort.
  2. In the Methods pane for the object, click the Name tab.

To sort methods by module:

  1. Select the object whose methods you want to sort.
  2. In the Methods pane for the object, click the Module tab.

To sort methods by subroutine name:

  1. Select the object whose methods you want to sort.
  2. In the Methods pane for the object, click the Subroutine tab.
