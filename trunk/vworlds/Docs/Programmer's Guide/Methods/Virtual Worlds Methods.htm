Virtual Worlds Methods

There are three types of methods in Virtual Worlds:

  • Script Methods
  • Inline Methods
  • System Methods

Script Methods

A script method is a subroutine or function written in Microsoft Visual Basic Scripting Edition© or Microsoft JScript©. Before you start to write the code for your method, consider whether the method should return a value. If the method does not need to return a value, write a subroutine to implement your method. If the method does need to return a value, write a function.

Once you have decided whether to implement your method as a subroutine or as a function, you can carry out the following four-step process in order to add the method to an object:

  1. Create a text file that contains one or more script subroutines or functions. Each of these subroutines and functions could potentially become a method on an object.
  2. Create a script module from this file.
  3. Create a method from the script module.
  4. Add the method to the appropriate object.

These steps are examined in detail in the topic Adding a Method from a Script Module.

Note that once a module is created from a text file containing script code, the actual code in the module is not updated unless you explicitly reload the module.  Simply changing code in the text file will not cause the module to be reloaded.

Inline Methods

An inline method is a type of method that is not associated with an external module. In contrast, script methods are associated with external modules. The inline method type is ideal for writing methods that can be completed within short code fragments. Unlike script methods, inline methods can be edited in the Inline Method Editor (one of the editors in the Object Explorer). Also, if a change is made to an inline method, the server and other clients will automatically be updated. This is not true of script methods.

All inline methods are associated with a single internal inline script module.

The topic Writing an Inline Method contains more information.

System Methods

All of the Virtual Worlds core objects and exemplars have native methods that are implemented in C++.  These methods cannot explicitly be changed by the user, but many of them can be overridden by re-implementing the methods as script or inline methods.
