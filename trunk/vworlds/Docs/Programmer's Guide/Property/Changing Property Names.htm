Changing Property Names

Editing property names can have one of the following results:

  • You cannot edit the name of a property that has been inherited from an exemplar. You must first copy the property (by editing its value) to create an object-local version of it, and then you will be able to change the property's name.
  • If you edit the name of a property that was inherited from an exemplar but has a new value, then you will actually create a new local version of that property, of the same type and value as the original but with your new name. The original exemplar property with the old name will not be deleted; it still remains in the object, unchanged.
  • If you edit the name of a property that exists only in the object instance, then you are in fact changing the name of that property. No property with the old name remains, it will be replaced by the property with the new name. However, be aware that renaming a property in this way can potentially break any references made to the property's old name.

To change a property's name:

  1. In the Lists pane, click the Author Tools tab, and then click Object Explorer.
  2. Right-click the property you want to change, and then click Edit Name.
    The name of the property will be highlighted.
  3. Type a new name for the property, and then press ENTER.
