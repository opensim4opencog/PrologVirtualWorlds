Adding a Property

In addition to changing the names and values of existing properties in an object, you can also add a new property to an object by using editors built into the Object Explorer, or using script in the Command Window. If you are more comfortable using Virtual Worlds' user interface than with writing script, use the Object Explorer to add a property. If you are comfortable writing script, however, you can use the Command Window to add a property.

To add a property to an object, using the Object Explorer:

  1. Right-click anywhere in the Properties pane of the Object Explorer.
  2. From the menu that appears, click New....
  3. In the New Property editor, do the following:
    • Enter a name for the property in the Property Name input box.
    • Select a type for the property in the Property Type selection box. If you select String, Boolean, or one of the four listed numeric types, the property is initialized with a default value, which you can change in the next step. If you select a type other than these, skip the following step.
    • Enter a new value, if desired, for the property in the Initial Value input box.
  4. [Optional] The default values of the Property Flags should be satisfactory for most properties (Copy On Get - checked, Not Remoted - unchecked, Not Logged - unchecked). If you uncheck Copy On Get, and make a copy of the object that contains this property, both objects share the same property. Any change in the property on one object affects the other object's property, as well. Checking Not Remoted means that changes to the property are not propogated to other clients. Checking Not Logged means that the property is not saved to the Virtual Worlds database.
  5. Click OK.

If the property you added is not a String, Boolean, Double, Float, Long, or Integer, it is initialized with a default value for that type. After you click OK, you can edit the object by clicking the Expand Dialog button that appears at the right end of the line containing the property. Depending on the type of property you chose, one of the following editors opens. For more information on a specific editor, choose one of the following links:

  • Vector Editor
  • Menu Editor
  • Property List Editor
  • Property Map Editor
  • Thing Selector

To add a property to an object, using the Command Window:

  • To add a property to a Thing object, use its AddProperty method:

    NewObj.AddProperty "Color"

You can, optionally, specify an initial value for the property and specify various security options when creating the property. Note that it is an error to add a property that is already present in an object or any of its exemplars.

Because you sometimes don't know whether a property is already present or not, you may have to check for a property's existence, and then create it if it does not already exist.

If NewObj.IsValidProperty ("Color") = False Then
   NewObj.AddProperty "Color", "Yellow"
Else
   World.Trace "Property already exists!"
End If

Here, the IsValidProperty method is used to determine whether the Color property exists on the object or on an exemplar in its inheritance chain.
