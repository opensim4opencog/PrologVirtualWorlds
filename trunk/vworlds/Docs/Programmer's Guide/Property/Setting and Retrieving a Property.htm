Setting and Retrieving a Property

Although you can use the Object Explorer to view or retrieve the values of properties on most objects in a world, the Virtual Worlds security model prevents users from changing object properties. If you are the owner of an object (or a wizard in the world), you can set a property by using the Object Explorer. You can also set or retrieve a property's value by using object.property notation in the Command Window.

To set the value of a property - Object Explorer:

  1. In the Object Explorer, select the object whose property you want to change.
  2. In the Properties pane, select the property, and then right-click it. Note: for security reasons, some properties cannot be edited. These properties do not have an Edit Value option, and have special icons signifying that they cannot be edited.
  3. From the menu that appears, click Edit Value.
  4. If the property is a numerical or string type, enter a new value for the property. If the property is Boolean, select True or False from the selection box. If the property is none of these types, an edit window of the appropriate type is displayed.

To set the value of a property - Command Window:

  • Type an assignment statement with this form into the Command Window:
    ObjectName.PropertyName     = NewValue

For example, the following assignment statement changes the Description property of an object named NewObj.

NewObj.Description = "This amulet makes you invisible"

You can also set the value of an added property using the same object.property notation. In the following example, the first line uses the AddProperty method of ThingColor property to NewObj. In the second line, the newly added property is set to the String value, "Green". 

NewObj.AddProperty "Color"
NewObj.Color = "Green"

To retrieve the value of an object's property - Command Window

  • For numeric, Boolean, or String properties, type an assignment statement of this form into the Command Window:
    Value = ObjectName.PropertyName

For example, this statement retrieves the value that is currently stored in the Color property on NewObj:

CurrentColor = NewObj.Color
  • If the property is Virtual Worlds core object (such as Vector, PropertyList, and others), include Set in the assigment statement, using an assignment statement of this form:
    Set Value = ObjectName.PropertyName

For example, the following statement retrieves the value of the Position property (of type Vector) on NewObj:

Set ObjLocation = NewObj.Position
