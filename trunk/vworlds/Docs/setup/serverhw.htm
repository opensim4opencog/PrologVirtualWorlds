Virtual Worlds Server Hardware and Software Requirements

Applications built on the Microsoft Virtual Worlds platform consist of client and server components. Both can run on the same machine, although this is not recommended for performance reasons. Multiple clients can run on the same machine. Each world runs on a single server, and each server can host multiple worlds.

Hardware requirements do vary greatly with the type of application and number of concurrent users so utilize these as recommended starting points.

Content associated with worlds needs to be accessible by Virtual Worlds Client applications either by a website or by a directory share (on an intranet).

Hardware Requirements, Preferred

Microsoft Virtual Worlds runs best with these preferred hardware requirements (or greater):

  • A Pentium II-350MHz (or greater) CPU
  • 128 MB of RAM
  • 50 MB of free disk space (more may be needed as more worlds are hosted)

Hardware Requirements, Minimum

Microsoft Virtual Worlds can run with these minimum hardware requirements:

  • A Pentium 266MHz CPU
  • 64 MB of RAM
  • 10 MB of free disk space (more will be needed as worlds are hosted - we recommend 50 MB)

Minimum Software Requirements

  • Microsoft Windows NT 4.0 (Service Pack 4), or Windows 2000.
  • Microsoft Web Publishing Wizard, version 1.52 (or greater).
  • ADS (Directory Services Software) - see the Virtual Worlds site, http://vworlds.research.microsoft.com.

To run the Server-Only installation (without running the graphical Virtual Worlds client), the server can use Windows NT 4.0 (Service Pack 4).
