Object Explorer Icons

Icons are grouped according to the pane in which they appear.

Object Explorer Tree Control (left pane)

Indicates that the object contains other objects. You can click this icon to expand the tree view.
Indicates that objects contained should all be visible in the tree view. You can click this icon to contract the tree view.

Object Explorer Properties (upper right pane)

A property inherited from an exemplar, that still has the exemplar's original value.
A property inherited from an exemplar, but with its value changed on the instance.
A property created on the selected object instance, that does not exist on the exemplar or on any other instance of that exemplar.
A read-only exemplar property. It cannot be changed.
A read-only instance property that overrides an exemplar property. It cannot be changed.
A read-only instance property. It cannot be changed.

Object Explorer Methods (lower right pane)

A method inherited from the exemplar that is unchanged on the instance.
An instance method that overrides an inherited exemplar method. An instance method overrides an exemplar method when it has the same name as the exemplar method, and has the same number and types of parameters.
A method that exists only in the object instance; it does not exist on the exemplar or on any other instance of that exemplar.
A read-only method inherited from the exemplar. It cannot be changed.
A read-only instance method that overrides an exemplar method. It cannot be changed.
A read-only instance method that exists only in the object instance. It cannot be changed.
