New Method Dialog Box (Object Explorer)

Use this dialog box to add a new method to the current object in the Object Explorer.

Add a new method to an object if you would like it to exhibit special behavior that Virtual Worlds does not provide. For example, you can program an object to be a bot, an object that behaves like a robot avatar under the computer's control, and can give help to users or participate in limited conversation.

Options

Object

This text displays the object to which the new method will be added, which is always the object currently selected in the Object Explorer.

Method Name

Enter the name of your new method here. Try to use a name suggestive of what the method actually does. For example, if you are adding a method that will make your avatar float into the air, name your method something like Float or Fly, not JoesMethod or Method22.

Module

Select the module that contains the code for your method from this drop-down menu. If you have added a module to the world, it will be listed in this menu; otherwise this menu will have only a single entry, Inline Module.

Subroutine

If you have added a module to the world, enter the name of the subroutine (or function) in that module. If you have not added a module to the world, this option is grayed out.

Execute on

Tell Virtual Worlds whether your method should be run on the client, on the server, or both, by selecting a value from this drop-down menu.

  • If you select Client, and if the method is invoked from a client, it runs only on that client, but if the method is invoked from the server, it runs remotely on all bystander clients.
  • If you select Server, and if the method is invoked from a client, it runs remotely on the server, but if the method is invoked from the server, it runs only on the same server.
  • If you select Client and Server, and if the method is invoked from a client, it is invoked both on that client and on the server, but if the method is invoked from the server, it runs remotely both on all bystander clients and also on the server.
