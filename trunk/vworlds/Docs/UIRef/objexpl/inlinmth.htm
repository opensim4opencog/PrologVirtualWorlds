Inline Method Editor

In the Inline Method Editor dialog box, you can write or edit the code for an inline method whose code is confined to a single sub or function.

Options

Object

This text displays the name of the object whose method you are adding or editing.

Inline Method

You can select one of the inline methods available in the world here. This menu lists all of the inline methods currently in the world.

Edit Name

Click this button to enter a new name for the method you are adding or editing.

New Method

Click this button to create a new inline method on the object being edited. (The name of this object appears under the Object heading of this window.) The new method is visible in the Methods pane of the Object Explorer.

Load Script

Click this button to load a script file from your hard disk. This makes it easy to edit a script in your favorite text editor, then load the script into Virtual Worlds.

Save to Script

Click this button to save the script displayed in the main pane of this dialog to a text file. After the script is saved, you can print the script, export it to another virtual world, or edit it in another application and load it back into Virtual Worlds with the Load Script button.

Delete

Click this button to completely delete the method you are editing from the virtual world, and optionally delete the menu items associated with it. After clicking this button, the method name is no longer displayed in the Methods pane of the Object Explorer.

Caution! After you delete a method from your virtual world, any methods or properties that call it will fail with an error.

Add Menu Item

Click this button to create a new menu item that calls the method you are editing. (The New Menu Item dialog box is displayed.)

Execute on

A menu appears when you click here; tell Virtual Worlds whether your method should be run on the client, on the server, or both by selecting a value from this menu.

  • If you select Client, then if the method is invoked from a client, it will be run only on that client, but if the method is invoked from the server, it will be run remotely on all bystander clients.
  • If you select Server, then if the method is invoked from a client, it will be run remotely on the server, but if the method is invoked from the server, it will be run only on the same server.
  • If you select Client and Server, then if the method is invoked from a client, it will be invoked both on that client and on the server, but if the method is invoked from the server, it will be run remotely both on all bystander clients and also on the server.
