Export As... Dialog Box (Virtual Worlds Server)

You can export the information for the world into a Microsoft Visual Basic Script© text file (extension .VBS). The script is created in the Worlds directory, using the filename that you enter. This script consists of a single subroutine that, when executed, reconstructs all of the objects in the world at the time the script was exported. The Export As... option of the Virtual Worlds Server's File menu (and the Export option) provides compatibility between different versions of Virtual Worlds software. If you save the world as a script file using one version of Virtual Worlds, you can load the world (by using the Import... menu option) with another version. This is not true for checkpoint files.

To export the world to a script file:

  1. In the File name entry box, type in the new script filename.
  2. Click Save.

After the world information is exported to a text file, you can restore that world by using the Import menu item of the File menu.
