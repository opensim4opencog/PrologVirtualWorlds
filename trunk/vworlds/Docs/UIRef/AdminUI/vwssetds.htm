Directory Server Address Dialog Box (Virtual Worlds Server)

Use this dialog box to set the URL (Universal Resource Locator) for the location of the Directory Services server. The Directory Services server, a computer running Microsoft Site Server©, lists all of the worlds that have been published to it. The Virtual Worlds Server application enables you to publish your world to the Directory Services server by using ADSI, the Active Directory Services Interface. This interface uses the Lightweight Directory Access Protocol, or LDAP, to enable client users to publish their worlds to the Directory Services server. Once your world is published to the Directory Services server, users in one virtual world can elect to visit another merely by clicking its name in the list of worlds that the Directory dialog box displays.

To use this protocol, you need to enter two pieces of information:

  • The computer name for the Directory Services server
  • The name of the Directory Services directory

For the example in the accompanying figure, the computer name is carlh2, and the Directory Services directory is carlhvworld. The last item, dynamic, indicates that any listing created by the Directory Services server will be stored dynamically (in memory), rather than on the client's hard disk.

To use the Directory Server Address dialog box:

  1. Type the name of the machine that is hosting the Directory Services server, and the name of the directory. The syntax for this line is LDAP://<computer name>/o=<directory>/ou=dynamic

    For example: 

    LDAP://DSServer/o=DSList/ou=dynamic
  2. Click Set.
  3. Check the first line of the Statistics pane in the Virtual Worlds Server application to see if your world is connected to the Directory Services server. If your world successfully connects, the status message reads, "Connected to DS". If not, the status message reads, "Not connected to DS".
