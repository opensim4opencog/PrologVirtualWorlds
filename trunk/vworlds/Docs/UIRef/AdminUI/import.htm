Import Script Wizard

You can export a virtual world into a Microsoft Visual Basic Script© file. By exporting the world, you can reconstruct the world and all the objects in it by importing the file.

Options

Enter your world name and server location

World name

Enter the "friendly" name for the world here. This can match the name the world had before, if you are restoring a virtual world. This is the name that is displayed in the Directory dialog box whenever people choose a world to visit, and the "Show World in Directory Services" option is turned on for your world on the server. The name can be up to 255 characters long.

World description

You can type a short description of your new world into this text box, so that people can decide whether they want to visit the world. The description is optional.

Server

Type the location of the Virtual Worlds server you plan to use into this text box. If the server you plan to use is the computer on which you are currently creating the new world, type localhost.

Setup your world administrator logon

Administrator name

Type the name you want to use for the administrative avatar of your new world into this text box.

Password

Type the password for your administrative avatar into this text box.

Confirm Password

Re-type your new password into this text box. We recommend you make your password easy to remember, so that you never lock yourself out of your world.

Enter the name of the exported database to import

Script filename

Enter the name of the exported world file here, with the pathname (if necessary). You can use Browse to find the file in another directory. Export files are Visual Basic files, and have a .VBS extension.

Import

Click to import the Visual Basic database file of the virtual world. The Script Events window shows the progress and status of the file.
