Approach Property Page

The approach distance is the amount of space between an avatar and an object when the avatar does an Approach command on that object. This is useful to set for large objects, tall objects, or wide objects, where the approaching avatar may get so close they cannot tell what the object is.

Property Page Options

Approach distance

When an avatar approaches an object, the avatar stops at a specific distance from the object. If your object is tall or wide, you may want to set a long distance, so the approaching avatars can see the entire object. If the object is very small, on the other hand, a short distance allows an avatar to move close and examine it.

The default approach distance is 4 units (each unit equals a meter). By default, two avatars who approach an object simultaneously are placed slightly apart from each other, the same distance from the object.

Random scatter distance

When avatars approach an object, they are stopped at the same distance each time. You can set a random placement from this object to scatter the approaching avatars, so they don't stack onto each other in the same spot. This is especially useful for portals, preventing the initial stacking of avatars when they approach the exit. The default random scatter distance is 1 unit (each unit equals a meter). If you click Approach to the front of the object, all avatars approach the object in the same area in the front, and you may experience crowding, but the random scatter distance may prevent stacking of the avatars.

Approach to the front of the object

This is on by default, so that avatars approach the front of the object at the set distance. You can turn it off so that avatars approach the object from all sides, but at the same distance.
